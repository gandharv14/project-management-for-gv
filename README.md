# Team Management App

A single-tenant team management app for projects, tasks, suggestions, recurring duties, and manager reporting.

## Stack

- Next.js App Router on Vercel
- Supabase Postgres and Realtime
- Auth0 with Labelbox SSO
- Tailwind CSS and shadcn/ui-style local components

## Local Setup

1. Copy `.env.example` to `.env.local`.
2. Fill in Auth0 and Supabase values.
3. Run the Supabase migration in `supabase/migrations`.
4. Start the app:

```bash
npm run dev
```

For local Auth0 testing, configure the Auth0 application with:

- Allowed Callback URLs: `http://localhost:3000/auth/callback`
- Allowed Logout URLs: `http://localhost:3000`
- Allowed Web Origins: `http://localhost:3000`
- Application Login URI: `http://localhost:3000/login`

For Vercel, add the deployed app origin to the same Auth0 application:

- Allowed Callback URLs: `https://<your-vercel-domain>/auth/callback`
- Allowed Logout URLs: `https://<your-vercel-domain>`
- Allowed Web Origins: `https://<your-vercel-domain>`
- Application Login URI: `https://<your-vercel-domain>/login`

Leave `APP_BASE_URL` unset unless every login starts and returns on one stable origin. With it unset, the Auth0 SDK infers the current request host, which avoids state-cookie mismatches across localhost, Vercel preview URLs, and production domains.

Leave `AUTH0_CONNECTION` empty unless the named Labelbox SSO connection is enabled for this Auth0 application. If it is set to a disabled or misspelled connection, Auth0 redirects back to `/auth/callback` with `invalid_request: the connection is not enabled`.

## Recurring Tasks

Recurring task instances are generated by `GET /api/cron/recurring`. Configure Vercel Cron to call it daily with:

```http
Authorization: Bearer $CRON_SECRET
```

The route generates concrete task rows from active recurring rules and advances each rule's `next_run_on`, preserving task completion history.
